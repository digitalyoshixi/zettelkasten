A usability inspection method that is guided by a set of recognized usability principles.
1. Use a list of evaluations
2. Brief experts to tell them what to do
3. Expert takes one pass to feel the product, then a second pass where they note each problem
4. Match each problem to a corresponding heuristic broken
5. Report each problem separately
6. Confer with other evaluators, and develop priorities for each problem
# Evaluations
1. Visibility of system status
2. [[Real World and System Synthesis]]
3. [[User Control and Fr`eedom]]
4. Consistency and standards
5. [[Error Prevention]]
6. Recognition rather than recall
7. Flexibility and efficiency of use
8. [[Minimalist Design]] & Aesthetic design
9. Help users recognize, diagnose and recover from errors
10. Help and documentation
